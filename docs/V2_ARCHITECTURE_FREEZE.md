# AISync Demo V2 - Architecture Freeze

Last updated: 2026-04-09
Status: Active doctrinal reference for V2

## Purpose

This document freezes the functional architecture, real scope, and base shell direction for AISync Demo V2 before any visual redesign, navigation refactor, or UX restructuring starts.

This is a doctrinal and structural freeze.

It does not authorize:
- broad refactors
- speculative architecture changes
- feature expansion
- rewriting validated product logic

## Product Reading

AISync must not be read as:
- a multi-chat app
- a PM tool with AI
- a decorative dashboard
- a polished chat interface

AISync must be read as:
- a control layer for AI-assisted work
- resumable work infrastructure
- a work system with structure, traceability, operational memory, human control, checkpoints, continuity, and resume capability

## Foundational Rule

AISync Demo V2 does not start from zero.

V2 preserves the valuable functional architecture already validated in the current product line and rebuilds the shell layer, navigation hierarchy, visual structure, and base UI language on top of that foundation.

The working rule is:
- preserve useful logic
- preserve validated flows
- rebuild shell and visual system deliberately
- avoid destructive rewrites

## Base Reference

For doctrinal and functional guidance, V2 takes as its base reference the consolidated state of AISync Demo V4.

V1 must remain intact as:
- operational reference
- stable backup
- non-destructive fallback

Work for V2 happens only in its independent project copy.

## Conceptual Sequence To Preserve

The core system reading remains:

1. Teams Map
2. Audit Log
3. Main Workspace

Interpretation:
- first orientation
- then traceability
- then execution

This sequence must continue informing shell decisions, module hierarchy, and how the product is explained visually.

## Frozen Functional Scope For Demo V2

Demo V2 is defined as a single autonomous local cell.

### 1. Workspace

The following capabilities remain mandatory:
- Review & Forward
- AUDIT AI ANSWER
- Save / Backup
- Save Version
- Saved Chats / Version History
- Resume Work
- Lock / Unlock as an operational control

### 2. Team Map / Tree

The following capabilities remain mandatory:
- MAP / TREE
- zoom
- pan
- Edit Team
- Add Team
- hierarchy reading
- Operational Elasticity View

### 3. Cross Verification

The following capabilities remain mandatory:
- entry from AUDIT AI ANSWER
- opening in a new window
- preserving the origin in its source workspace
- Sub-Manager as the primary surface
- automatic worker participation
- CHOOSE THIS ANSWER
- return into the real destination chat

### 4. Documentation Mode

Documentation Mode remains part of V2.

Rules:
- it is not removed
- it remains under selective simplification review
- it must not be amputated by reflex
- whatever still provides real value for local memory, local traceability, and autonomous cell operation should be preserved

## Out Of Scope In This Phase

The following are explicitly outside the scope of this freeze stage and the immediate V2 shell phase:
- real interaction between cells
- multi-cell operational architecture
- communication between cell managers
- inter-cell traceability
- any new feature that expands AISync beyond one autonomous local cell
- deep logic refactors not required for shell isolation or UX restructuring

## Architecture Decision For V2

### Preserve

The following are preserved:
- general functional architecture
- validated flows
- operational memory
- Audit Log as an entry gate
- versioning and resume work
- lock as operational control
- Team Map as orientation surface
- Cross Verification as a distinctive capability
- Documentation Mode as the documentary base of the system

### Rebuild

The following are intentionally rebuilt:
- general shell
- primary desktop navigation
- page headers
- visual hierarchy
- visual density
- interface typography
- base panel patterns
- lateral detail pattern
- consistency of buttons, states, filters, and listings
- node abstraction language in MAP / TREE
- the full visual language of the demo

## Base Shell Definition For V2

AISync Demo V2 will work on a desktop-first shell with four structural layers:

### A. Global Left Navigation

Purpose:
- primary system modules
- stable navigation spine

### B. Contextual Lateral Area

Purpose:
- team context
- workspace context
- active view context when applicable

### C. Main Canvas

Purpose:
- operational content of the current module

### D. Optional Right Detail / Inspector Panel

Purpose:
- metadata
- audit context
- selected object detail
- contextual properties

## Navigation Rule

The current bottom navigation must not remain the primary desktop pattern.

It may survive only as:
- mobile adaptation
- fallback navigation pattern where appropriate

## Primary Modules For The V2 Cell

The primary modules of the autonomous cell are:
- Cell Map
- Audit Log
- Workspace
- Documentation

Cross Verification remains a strong contextual module, not the primary shell axis.

## Implementation Rule After This Freeze

This freeze authorizes future work only under the following conditions:
- do not redesign full screens impulsively
- do not rebuild components without scope justification
- do not touch validated logic unless shell isolation requires it
- do not introduce new features disguised as cleanup
- do not reopen product readings already frozen here

## Non-Regression Criteria

The following criteria remain mandatory:
- do not break closed V1 or V4 criteria
- do not touch the live V1 folder
- do not reframe AISync as a multi-chat product
- do not turn AISync into a generic startup dashboard
- do not flatten Documentation Mode into a poor file explorer
- do not reduce Team Map to decoration
- do not reduce Audit Log to a dry forensic log

## Operational Use Of This Document

This file must be treated as a required reference before any OE that affects:
- shell
- navigation
- module hierarchy
- page framing
- cross-module information architecture
- Documentation Mode simplification decisions

If a later OE conflicts with this document, the OE must explicitly justify the exception instead of silently overriding the freeze.
